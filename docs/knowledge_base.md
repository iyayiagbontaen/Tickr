# Sample Knowledge Base

Replace this file with text your app should use. You can paste class notes, a company policy, product descriptions, assignment instructions, or a short memo.

For the starter demo, the app should know the following:

APIs are often the best first move for early AI applications because they let a team test a product idea without running model infrastructure. They are fast to integrate and shift reliability, scaling, and model maintenance to the provider.

Renting GPUs or hosting an open model can make sense when usage is large and predictable, when API margins become too expensive, when latency or data-control requirements are unusually strict, or when the model itself becomes part of the product's defensibility.

RAG is useful when an application must answer from a specific set of documents or current facts. It retrieves relevant source text at runtime and adds that text to the model prompt.

Fine-tuning is usually unnecessary for a short prototype. It is more useful when a team has many examples and needs the model to repeatedly match a specific style, format, or domain behavior.
